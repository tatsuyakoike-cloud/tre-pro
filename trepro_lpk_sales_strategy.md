# トレプロ営業方針・LPKユニットエコノミクス改善計画

> Web公開用 Markdown  
> 目的：7月・8月で、オフライン営業とオンライン営業の2軸体制を立ち上げ、Q4に向けてLPKの回復モデルを作る。

---

## 1. 全体方針

### 方針

今後の営業体制は、以下の2軸で進める。

| 軸 | 目的 | 主な役割 |
|---|---|---|
| オフライン営業 | 短期売上の創出 | 代理店、紹介、地方営業、飛び込み、訪問、クロージング |
| オンライン営業 | 既存リストの再活用と仕組み化 | LPKリスト整理、掘り起こし、メール配信、架電検証、ナーチャリング |

### なぜ2軸で進めるのか

- LP広告の獲得効率が悪化しており、広告単体での回復には限界がある。
- 過去に接点を持ったリストが残っているが、十分に営業資産として活用できていない。
- 代理店・紹介・地方営業は短期の受注機会を作れる一方で、属人的になりやすい。
- オンライン側で、リスト・配信・架電・商談化の流れを作ることで、再現性のある営業基盤を作れる。
- 7月・8月は、短期売上を作りながら、Q4に向けて継続的に商談が生まれる状態を作る期間とする。

---

## 2. 現状認識

### 現状の課題

- LPKの獲得コストが上がっている。
- 顧客の継続月数が以前より短くなり、LTVが下がっている。
- 以前よりも、広告費をかけても利益が残りづらい構造になっている。
- 現状のままでは、LPK施策が撤退ラインに入る可能性がある。
- そのため、7月・8月で既存リストの掘り起こしと営業体制の再設計を進める必要がある。

## 3. LPKの数値前提

### 今回採用する前提

| 項目 | 数値 | 補足 |
|---|---:|---|
| 総獲得コスト | 800万円 | LP広告・営業人件費・外注費・ツール費などを含めた施策全体のコスト |
| 目標倍率 | 3.0倍以上 | 撤退ラインを超える最低基準 |
| 必要LTV合計 | 2,400万円 | 800万円 × 3.0倍 |
| 1社あたりLTV目安 | 330万円 | 月額単価 × 想定継続月数で算出する期待値 |
| 現状例のLTV合計 | 900万円 | 330万円 + 240万円 + 330万円 |
| 現状例の倍率 | 1.13倍 | 900万円 ÷ 800万円 |

### 注意点

- 800万円は、1社あたりのCACではなく、施策全体の総獲得コスト。
- 成約数が増えるほど、1社あたりCACは下がる。
- 3倍ラインを見る場合は、施策全体で `LTV合計 ÷ 総獲得コスト` を確認する。

## 6. 営業戦略の2軸

---

### 6-1. オフライン営業

#### 目的

短期の商談・受注を作る。

#### 中心テーマ

| テーマ | 内容 |
|---|---|
| 代理店・紹介 | 既存の代理店・紹介ルートを活用し、短期で受注可能性のある案件を進める |
| 地方営業・飛び込み | 地方企業への直接営業、訪問、飛び込みを実施する |
| クロージング | 受注確度の高い案件を個別対応し、早期に契約へ進める |
| 価格検証 | 料金がボトルネックの案件に対し、値引き・初月無料などのABテストを行う |

#### 実行内容

- 代理店・紹介経由のホットリードを整理する。
- 地方案件・飛び込み対象を抽出する。
- 高確度案件は訪問・個別提案・クロージングを優先する。
- 料金が主な失注理由になっている案件では、値引きあり提案を限定的に検証する。
- 値引きは通常化せず、受注率・LTV・CACへの影響を確認する。

#### 見るKPI

| KPI | 確認内容 |
|---|---|
| 接触数 | 代理店・紹介・地方営業で接触した企業数 |
| 商談数 | 実際に商談化した件数 |
| 受注数 | 契約に至った件数 |
| 平均LTV | 受注1社あたりの期待LTV |
| 値引き有無別受注率 | 値引きあり・なしで受注率に差が出るか |
| 施策別CAC | オフライン施策にかかったコストと受注数の関係 |

---

### 6-2. オンライン営業

#### 目的

既存リストを再活用し、低コストで商談機会を作る。

#### 中心テーマ

| テーマ | 内容 |
|---|---|
| リスト整理 | LPK・過去商談リストを確度別に整理する |
| 掘り起こし | メール配信と架電で、過去接点から再商談化を狙う |
| ナーチャリング | ホワイトペーパー、ウェビナー、LINE等で継続接点を作る |
| データ化 | 配信反応、架電結果、商談化率を記録し、次のKPIを作る |
| 仕組み化 | CRM・SFA・MAへ接続できる運用ルールを整える |

#### 実行内容

- LPKリストをTier1〜4に分類する。
- Tierごとに送付物を変える。
- メール配信後、開封・クリック・返信を確認する。
- 反応企業を中心に架電する。
- 最初の検証として、100件程度の架電データを作る。
- 反応が薄い層にはホワイトペーパーやウェビナーで接点を継続する。
- 今回の掘り起こし施策は、過去の商談受注率を前提にせず、新しく基準値を作る。

#### 見るKPI

| KPI | 確認内容 |
|---|---|
| 配信数 | Tier別のメール送付数 |
| 開封数 | メールが見られた件数 |
| クリック数 | 資料・リンクがクリックされた件数 |
| 返信数 | 反応があった件数 |
| 架電数 | 配信後に架電した件数 |
| 接続数 | 電話がつながった件数 |
| アポ数 | アポイント化した件数 |
| 商談数 | 実際に商談化した件数 |
| 受注数 | 契約に至った件数 |
| 施策別CAC | オンライン施策にかかったコストと受注数の関係 |

## 7. Tier定義

| Tier | 対象 | 目的 | 主なアクション |
|---|---|---|---|
| Tier 1 | 成約角度40%以上 | 商談化・クロージング | アポ打診、事例集、料金表、架電、訪問 |
| Tier 2 | 成約角度20% | 掘り起こし・再商談化 | 会社資料、アポ打診、反応後架電 |
| Tier 3 | 成約角度10% | 中長期ナーチャリング | ホワイトペーパー、業界情報、ウェビナー案内 |
| Tier 4 | それ以外 | ステイ | 原則アプローチしない |

### 補足

- 40%は、課題とネクストアクションが明確な状態とする。
- 20%は、過去接点があり、再商談化の余地がある状態とする。
- 10%は、すぐに商談化はしないが、情報提供を続ける価値がある状態とする。
- Tier4は、明確NG、電話NG、対象外、情報不足などを含む。

## 8. Tier別アプローチ

| Tier | 送付内容 | 架電方針 | 次の状態 |
|---|---|---|---|
| Tier 1 | 事例集、料金表、アポ打診 | 送付後に優先架電 | 商談・クロージング |
| Tier 2 | 会社資料、アポ打診 | 反応企業を中心に架電 | 商談化、またはTier1昇格 |
| Tier 3 | ホワイトペーパー、業界情報 | クリック・返信があれば架電 | Tier2昇格 |
| Tier 4 | 送付しない | 原則架電しない | ステイ |

## 10. 価格調整・ABテスト

### 目的

料金がボトルネックになっている案件で、価格調整が受注率に与える影響を確認する。

### 実施方針

| 項目 | 内容 |
|---|---|
| 対象 | 代理店・地方営業経由の高温度案件 |
| 検証内容 | 値引きあり提案と通常提案の比較 |
| 施策例 | 初月無料、期間限定、先着枠、地方限定条件 |
| 比較指標 | 受注率、平均単価、LTV、CAC、リードタイム |
| 注意点 | 値引きは通常化せず、検証施策として扱う |

## 11. CRM・ナーチャリング

### 方針

メール配信だけで終わらせず、継続的に接点を持つ導線を作る。

### 施策

| 施策 | 目的 |
|---|---|
| メール配信 | Tier別に接点を再開する |
| ホワイトペーパー | 温度感の低い層に情報提供する |
| ウェビナー | 採用広報・SNS採用の考え方をまとめて伝える |
| LINEコミュニティ | 継続的にSNS採用の情報を届ける |
| CRM/SFA/MA | 配信・架電・商談・受注を一元管理する |
