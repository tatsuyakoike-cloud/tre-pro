# トレプロ営業方針改善計画

## 1. 全体方針

今後の営業活動は、**オフライン営業**と**オンライン営業**の2軸で進める。

| 領域 | 役割 | 目的 |
|---|---|---|
| オフライン営業 | 代理店・紹介・地方営業・飛び込み・訪問・クロージング | 短期で商談と受注を作る |
| オンライン営業 | LP経由リストの整理・掘り起こし・配信・架電・ナーチャリング | 既存リストを営業資産化し、再現性のある商談創出を作る |

この2軸で進める理由は、LP広告の獲得効率が悪化している中で、広告単体の改善だけに依存せず、**短期の受注創出**と**中長期の営業基盤づくり**を同時に進める必要があるためである。

7月・8月は、Q4に向けて以下の状態を作る。

| 項目 | 8月末時点で作る状態 |
|---|---|
| 営業体制 | オフライン営業とオンライン営業の役割が明確になっている |
| LP経由 | 既存リストから商談・受注が生まれている |
| 数値管理 | 成約数、LTV合計、総獲得コスト、倍率を確認できる |
| リスト管理 | 確度別・Tier別に営業対象が整理されている |
| ナーチャリング | メール、ホワイトペーパー、ウェビナー、LINE等で継続接点を作れている |
| Q4準備 | 9月以降、CRM・SFA・MAへ接続できる運用ルールが整理されている |

---

## 2. 現状認識

### 2-1. 現状の課題

| 課題 | 内容 |
|---|---|
| LP経由の獲得コスト悪化 | 広告費をかけても、以前より効率よく顧客獲得できていない |
| LTVの低下 | 顧客の継続期間が短くなり、1社あたりの将来売上が下がっている |
| 広告依存の限界 | 広告改善だけで短期的に大きく回復する見込みが立ちづらい |
| 既存リストの未活用 | 過去接点・商談履歴がある企業を十分に掘り起こせていない |
| 営業ルールの未整備 | 確度、優先順位、失注理由、次回アクションの管理が統一されていない |

### 2-2. 今回の対応方針

- オフライン営業では、代理店・紹介・地方営業・飛び込みを活用し、短期の受注機会を作る。
- オンライン営業では、LP経由の既存リストを整理し、掘り起こし施策を新たに企画・実行する。
- これまでの商談受注率を前提に置きすぎず、今回の掘り起こし施策で新しい基準値を作る。
- 施策ごとに、成約数・LTV合計・総獲得コスト・倍率を確認する。

## 3. LP経由の数値前提

### 3-1. 数値の位置づけ

以下の数字は、現時点での**仮置き**である。  
確定値ではなく、シミュレーション上の目標値として、まずはこの水準を目指す。

| 項目 | 仮置き数値 | 補足 |
|---|---:|---|
| 総獲得コスト | 800万円 | LP広告・営業人件費・外注費・ツール費などを含めた施策全体のコスト |
| 目標倍率 | 3.0倍以上 | 撤退ラインを超える最低基準 |
| 必要LTV合計 | 2,400万円 | 800万円 × 3.0倍 |
| 1社あたりLTV目安 | 330万円 | 月額単価 × 想定継続月数で算出する期待値 |
| 現状例のLTV合計 | 900万円 | 330万円 + 240万円 + 330万円 |
| 現状例の倍率 | 1.13倍 | 900万円 ÷ 800万円 |

### 3-2. 補足

- 800万円は、1社あたりのCACではなく、施策全体の総獲得コストとして仮置きする。
- 2,400万円は、総獲得コスト800万円に対して3倍を達成するために必要なLTV合計。
- 330万円は、1社あたりのLTV目安として置く。
- 900万円は、例として3社分のLTVを合計した数字。
- 今後、実績が出次第、総獲得コスト・LTV・必要成約数は更新する。

## 5. 営業戦略の2軸

### 5-1. オフライン営業

#### 目的

短期の商談・受注を作る。

#### 重点テーマ

| テーマ | 内容 |
|---|---|
| 代理店・紹介 | 既存の代理店・紹介ルートを活用し、受注可能性の高い案件を進める |
| 地方営業・飛び込み | 地方企業への直接営業、訪問、飛び込みを実施する |
| クロージング | 高確度案件を個別対応し、早期に契約へ進める |
| 価格検証 | 料金がボトルネックの案件で、値引き・初月無料等を限定検証する |

#### 実行内容

- 代理店・紹介経由のホットリードを整理する。
- 地方案件・飛び込み対象を抽出する。
- 高確度案件は訪問・個別提案・クロージングを優先する。
- 料金が主な失注理由になっている案件では、値引きあり提案を限定的に検証する。
- 値引きは通常化せず、受注率・LTV・CACへの影響を確認する。

#### 見るKPI

| KPI | 確認内容 |
|---|---|
| 接触数 | 代理店・紹介・地方営業で接触した企業数 |
| 商談数 | 実際に商談化した件数 |
| 受注数 | 契約に至った件数 |
| 平均LTV | 受注1社あたりの期待LTV |
| 値引き有無別受注率 | 値引きあり・なしで受注率に差が出るか |
| 施策別CAC | 施策にかかったコストと受注数の関係 |

## 6. Tier定義

| Tier | 対象 | 目的 | 主なアクション |
|---|---|---|---|
| Tier 1 | 成約角度40%以上 | 商談化・クロージング | アポ打診、事例集、料金表、架電、訪問 |
| Tier 2 | 成約角度20% | 掘り起こし・再商談化 | 会社資料、アポ打診、反応後架電 |
| Tier 3 | 成約角度10% | 中長期ナーチャリング | ホワイトペーパー、業界情報、ウェビナー案内 |
| Tier 4 | それ以外 | ステイ | 原則アプローチしない |

### 補足

| 確度 | 定義 |
|---:|---|
| 40%以上 | 課題とネクストアクションが明確な状態 |
| 20% | 過去接点があり、再商談化の余地がある状態 |
| 10% | すぐに商談化はしないが、情報提供を続ける価値がある状態 |
| その他 | 明確NG、電話NG、対象外、情報不足など |

---

## 7. Tier別アプローチ

| Tier | 送付内容 | 架電方針 | 次の状態 |
|---|---|---|---|
| Tier 1 | 事例集、料金表、アポ打診 | 送付後に優先架電 | 商談・クロージング |
| Tier 2 | 会社資料、アポ打診 | 反応企業を中心に架電 | 商談化、またはTier1昇格 |
| Tier 3 | ホワイトペーパー、業界情報 | クリック・返信があれば架電 | Tier2昇格 |
| Tier 4 | 送付しない | 原則架電しない | ステイ |

## 9. 価格調整・ABテスト

### 目的

料金がボトルネックになっている案件で、価格調整が受注率に与える影響を確認する。

### 実施方針

| 項目 | 内容 |
|---|---|
| 対象 | 代理店・地方営業経由の高温度案件 |
| 検証内容 | 値引きあり提案と通常提案の比較 |
| 施策例 | 初月無料、期間限定、先着枠、地方限定条件 |
| 比較指標 | 受注率、平均単価、LTV、CAC、リードタイム |
| 注意点 | 値引きは通常化せず、検証施策として扱う |

## 10. CRM・ナーチャリング

### 方針

メール配信だけで終わらせず、継続的に接点を持つ導線を作る。

### 施策

| 施策 | 目的 |
|---|---|
| メール配信 | Tier別に接点を再開する |
| ホワイトペーパー | 温度感の低い層に情報提供する |
| ウェビナー | 採用広報・SNS採用の考え方をまとめて伝える |
| LINEコミュニティ | 継続的にSNS採用の情報を届ける |
| CRM/SFA/MA | 配信・架電・商談・受注を一元管理する |

## 11. Webサイト公開時の表示方針

### 表示ルール

- 無理なフローチャートは使わない。
- 数式説明よりも、シミュレーション表とKPI表を中心にする。
- スマホでも見やすいように、表はカード化または横スクロール対応にする。
- セクション全体が右寄り・左寄りにならないよう、中央配置を前提にする。
- オフライン営業とオンライン営業の2軸は、PCでは横並び、スマホでは縦並びにする。
